# CHAT 提示词框架

## 概念

```
Character 角色：指定角色信息
History 背景：提供历史信息和背景知识
Ambition 目标：描述希望实现的目标
Task 任务：明确具体的任务或行动
```

CHAT 框架集中于角色、背景、目标和任务四个核心部分，为用户与大模型的深度交互提供了全面的指导。

- **角色 (Character)**：为大模型提供用户身份和角色信息
- **背景 (History)**：提供与当前问题相关的历史信息和背景知识
- **目标 (Ambition)**：描述希望实现的长期或短期目标
- **任务 (Task)**：明确希望大模型执行的具体任务或行动

## 示例

**测试用例：Web 测试用例编写**

```
Character 角色：资深 web 测试人员
History 背景：项目需要完整的测试用例来保证质量，测试用例需要覆盖正向和反向场景
Ambition 目标：确保项目质量，提供全面的测试覆盖
Task 任务：收集项目需求，进行需求分析，编写 markdown 格式的测试用例
```

**Prompt**：你是一名资深 web 测试人员，项目需要完整的测试用例来保证质量，测试用例需要覆盖正向和反向场景。你的目标是确保项目质量，提供全面的测试覆盖。你的任务是先收集项目的需求，进行需求分析后，再编写 markdown 格式的测试用例，包含正向和反向场景。

## 适用场景

- 需要明确角色定位的任务
- 有复杂背景信息的场景
- 目标和任务都需要清晰表达的工作

## 优势

- 四要素覆盖全面，结构完整
- 强调角色和背景，提供充分上下文
- 适合需要深度交互的复杂任务
