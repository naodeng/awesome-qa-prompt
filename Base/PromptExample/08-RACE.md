# RACE 框架

## 概念

```
R 背景：设置讨论的舞台或背景。 
A 行动：描述您想要做什么。 
C 结果：描述期望的结果。 
E 示例：举一个例子来说明你的观点。
```

RACE 是一个用于创建提示的框架，它旨在提供结构和明确性。

- **背景 (Role/Background)**：设置讨论的舞台或背景
- **行动 (Action)**：描述想要做什么，明确请求或目标
- **结果 (Result/Context)**：描述期望的结果
- **示例 (Example)**：提供一个例子来说明观点

## 示例

**测试用例：Web 测试用例编写**

```
R 背景：作为资深 web 测试人员，需要为项目编写完整的测试用例
A 行动：收集需求、分析需求、编写测试用例
C 结果：得到一份包含正向和反向场景的 markdown 格式测试用例文档
E 示例：参考标准的测试用例模板，如登录功能测试用例
```

**Prompt**：你是一个资深 web 测试人员，需要为项目编写完整的测试用例。你的任务是先收集项目的需求，进行需求分析后，再编写测试用例。期望得到一份包含正向和反向场景的 markdown 格式测试用例文档。你可以参考标准的测试用例模板，如登录功能测试用例。

## 适用场景

- 需要提供背景和示例的任务
- 对结果有明确期望的场景
- 需要参考案例的工作

## 优势

- 包含示例要素，提供参考方向
- 结构清晰，易于理解
- 适合需要借鉴案例的任务
