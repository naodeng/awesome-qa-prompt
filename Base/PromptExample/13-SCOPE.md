# SCOPE 框架

## 概念

```
Scenario 情境：提供问题或任务发生的上下文和背景
Complications 复杂情况：描述问题中存在的难点或复杂因素
Objective 目标：明确应该达到的目标或预期效果
Plan 计划：一个或多个具体的行动方案
Evaluation 评估：对实施效果进行的考核或分析
```

SCOPE 框架由五个元素组成，是一个全面且具体的问题解决和任务管理框架。

- **情境 (Scenario)**：提供问题或任务发生的上下文和背景
- **复杂情况 (Complications)**：描述问题中存在的难点或复杂因素
- **目标 (Objective)**：明确应该达到的目标或预期效果
- **计划 (Plan)**：具体的行动方案
- **评估 (Evaluation)**：对实施效果进行的考核或分析

## 示例

**测试用例：Web 测试用例编写**

```
Scenario 情境：项目功能复杂，涉及多个模块交互，需要编写全面的测试用例
Complications 复杂情况：需求文档不够详细，部分功能点存在歧义，时间紧迫
Objective 目标：在一周内完成测试用例编写，覆盖率达到 90% 以上
Plan 计划：
1）与产品经理沟通澄清需求
2）梳理功能模块和交互流程
3）设计测试场景（正向、反向、边界）
4）编写 markdown 格式测试用例
5）组织用例评审
Evaluation 评估：通过用例评审，检查覆盖率，根据反馈优化用例
```

**Prompt**：项目功能复杂，涉及多个模块交互，需要编写全面的测试用例。当前面临的挑战是需求文档不够详细，部分功能点存在歧义，时间紧迫。你的目标是在一周内完成测试用例编写，覆盖率达到 90% 以上。计划如下：1）与产品经理沟通澄清需求；2）梳理功能模块和交互流程；3）设计测试场景（正向、反向、边界）；4）编写 markdown 格式测试用例；5）组织用例评审。最后通过用例评审，检查覆盖率，根据反馈优化用例。

## 适用场景

- 复杂的问题解决场景
- 需要考虑多种因素的任务
- 需要评估和优化的项目

## 优势

- 五要素覆盖全面，包含评估环节
- 强调复杂情况和计划，实用性强
- 适合复杂的项目管理和问题解决
