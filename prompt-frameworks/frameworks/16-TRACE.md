# TRACE 框架

## 概念

```
Task 任务：定义具体任务。 
Request 请求：描述您的请求。 
Action 行动：说明您需要采取的行动。 
Context 语境：提供背景或情况。 
Example 示例：举一个例子来说明你的观点。
```

TRACE 框架是一个用于创建和组织与 ChatGPT 交互的方法，它将交互细分为五个核心部分。

- **任务 (Task)**：定义要解决的特定任务或问题
- **请求 (Request)**：明确想要 ChatGPT 完成的具体请求
- **行动 (Action)**：描述应采取的步骤或操作
- **上下文 (Context)**：提供与请求相关的背景信息
- **示例 (Example)**：提供示例帮助理解请求和期望结果

## 示例

**测试用例：Web 测试用例编写**

```
Task 任务：编写项目的测试用例
Request 请求：请编写一份包含正向和反向场景的测试用例文档
Action 行动：收集项目需求，进行需求分析，编写测试用例
Context 语境：作为资深 web 测试人员，测试用例需要确保项目质量，格式为 markdown
Example 示例：参考标准的测试用例模板，如用户登录、数据验证等场景
```

**Prompt**：你的任务是作为一个资深 web 测试人员，编写一个项目的测试用例。请编写一份包含正向和反向场景的测试用例文档。你需要先收集项目的需求，进行需求分析，然后编写测试用例。测试用例需要确保项目质量，格式为 markdown。你可以参考标准的测试用例模板，如用户登录、数据验证等场景。

## 适用场景

- 需要明确任务和请求的场景
- 需要提供上下文和示例的工作
- 复杂的多步骤任务

## 优势

- 五要素覆盖全面，结构完整
- 包含示例要素，提供参考方向
- 适合需要详细指导的任务
