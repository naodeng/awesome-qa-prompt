# CARE 框架

## 概念

```
Context 上下文指导：提供操作的背景信息
Action 行动：描述要进行的具体活动
Result 结果：描述期望得到的效果或反馈
Example 示例：提供使用提示词的实例
```

CARE 框架强调四个关键要素：上下文指导、行动、结果和示例。

- **上下文指导 (Context)**：提供操作的背景信息，帮助大模型了解环境或情境
- **行动 (Action)**：描述要进行的具体活动
- **结果 (Result)**：描述完成活动后期望得到的效果或反馈
- **示例 (Example)**：提供使用提示词的实例，帮助理解如何正确交互

## 示例

**测试用例：Web 测试用例编写**

```
Context 上下文指导：作为资深 web 测试人员，需要为项目编写测试用例，确保项目质量
Action 行动：收集项目需求，进行需求分析，编写包含正向和反向场景的测试用例
Result 结果：得到一份完整的 markdown 格式测试用例文档，覆盖所有关键功能点
Example 示例：参考标准测试用例模板，如用户登录测试用例（正向：正确用户名密码；反向：错误密码、空输入等）
```

**Prompt**：作为资深 web 测试人员，你需要为项目编写测试用例，确保项目质量。你的任务是先收集项目的需求，进行需求分析，然后编写包含正向和反向场景的测试用例。期望得到一份完整的 markdown 格式测试用例文档，覆盖所有关键功能点。你可以参考标准测试用例模板，如用户登录测试用例（正向：正确用户名密码；反向：错误密码、空输入等）。

## 适用场景

- 需要提供完整上下文的任务
- 需要示例参考的场景
- 对结果有明确期望的工作

## 优势

- 包含示例要素，提供参考方向
- 强调上下文和结果，目标明确
- 适合需要具体指导的任务
