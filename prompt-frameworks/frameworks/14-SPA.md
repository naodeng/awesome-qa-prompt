# SPA 框架

## 概念

```
Scenario 情境：给出问题发生的上下文
Problem 问题：需要解决的核心难题或挑战
Action 行动：为解决问题需要采取的步骤或措施
Result 结果：行动后应达到的目标或效果
```

SPA 框架是一个用于解决问题和任务分配的综合框架。

- **情境 (Scenario)**：给出问题发生的上下文
- **问题 (Problem)**：需要解决的核心难题或挑战
- **行动 (Action)**：为解决问题需要采取的步骤或措施
- **结果 (Result)**：行动后应达到的目标或效果

## 示例

**测试用例：Web 测试用例编写**

```
Scenario 情境：项目即将上线，但测试用例还未编写完成
Problem 问题：如何快速高效地编写出高质量的测试用例，确保项目按时上线？
Action 行动：
1）优先梳理核心功能和高风险模块
2）采用测试设计方法（等价类、边界值）快速设计用例
3）编写 markdown 格式测试用例，包含正向和反向场景
4）与开发团队同步，确保用例可执行
Result 结果：在 3 天内完成核心功能测试用例编写，覆盖率达到 80%，确保项目按时上线
```

**Prompt**：项目即将上线，但测试用例还未编写完成。你面临的问题是如何快速高效地编写出高质量的测试用例，确保项目按时上线。作为资深 web 测试人员，你需要：1）优先梳理核心功能和高风险模块；2）采用测试设计方法（等价类、边界值）快速设计用例；3）编写 markdown 格式测试用例，包含正向和反向场景；4）与开发团队同步，确保用例可执行。期望在 3 天内完成核心功能测试用例编写，覆盖率达到 80%，确保项目按时上线。

## 适用场景

- 问题导向的任务
- 需要快速解决问题的场景
- 有明确结果要求的工作

## 优势

- 四要素聚焦问题解决
- 强调问题和行动，目标明确
- 适合紧急或问题驱动的任务
