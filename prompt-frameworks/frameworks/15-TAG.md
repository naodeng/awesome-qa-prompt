# TAG 框架

## 概念

```
Task 任务：定义具体任务。 
Action 行动：描述需要做什么。 
Goal 目标：解释最终目标。
```

TAG 框架是个简洁的框架，实际是 CHAT 模型的 AT 部分的细化，对场景依赖不明显的通用任务比较好，优点是简洁快速。

- **任务 (Task)**：定义需要完成的具体工作内容
- **行动 (Action)**：描述为了完成任务需要做什么
- **目标 (Goal)**：解释完成任务最终要达到的目的或效果

## 示例

**测试用例：Web 测试用例编写**

```
Task 任务：编写项目的测试用例
Action 行动：收集项目需求，进行需求分析，编写测试用例
Goal 目标：完成一份包含正向和反向场景的 markdown 格式测试用例文档
```

**Prompt**：你的任务是作为一个资深 web 测试人员，编写一个项目的测试用例。首先收集项目的需求，进行需求分析，然后开始编写测试用例。最终目标是完成一份包含正向和反向场景的 markdown 格式测试用例文档。

## 适用场景

- 简单直接的任务
- 不需要复杂背景的场景
- 快速构建提示词的情况

## 优势

- 结构简洁，易于理解和使用
- 三要素覆盖任务核心
- 适合快速执行的任务
