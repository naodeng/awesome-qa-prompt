# 自动化测试提示词框架集合

本目录包含了20个不同框架结构的自动化测试提示词，每个框架都提供完整版和轻量版两个版本。

## 🎯 适用场景

自动化测试是现代软件开发中不可或缺的重要环节，本框架集合适用于：

- **Web自动化测试**：UI自动化、功能自动化、回归测试
- **API自动化测试**：接口测试、集成测试、契约测试
- **移动端自动化测试**：Android/iOS应用自动化测试
- **数据库自动化测试**：数据验证、性能测试、一致性测试
- **CI/CD集成**：持续集成中的自动化测试策略

## 📋 核心输出内容

每个框架都会输出包含以下内容的自动化测试方案：

1. **方案概述**：自动化测试目标、范围、策略
2. **技术架构设计**：技术栈选择、架构设计、工具选型
3. **自动化测试策略**：测试分层、测试类型、覆盖策略
4. **框架设计详情**：框架结构、设计模式、最佳实践
5. **测试用例设计**：用例结构、数据驱动、参数化
6. **CI/CD集成方案**：持续集成、自动化部署、测试流水线
7. **测试报告和监控**：报告生成、结果分析、监控告警
8. **维护和优化策略**：框架维护、性能优化、扩展策略

## 📚 框架列表

### 基础结构框架
- **APE** (Action Purpose Expect) - 行动目的期望框架
- **BROKE** (Background Role Objectives Key Result Evolve) - 背景角色目标关键结果演进框架
- **CHAT** (Character History Ambition Task) - 角色历史抱负任务框架
- **CRISPE** (Capacity Role Insight Statement Personality Experiment) - 能力角色洞察陈述个性实验框架
- **CARE** (Context Action Result Example) - 上下文行动结果示例框架

### 目标导向框架
- **COAST** (Context Objective Action Support Technology) - 上下文目标行动支持技术框架
- **CREATE** (Clarity Relevant info Examples Avoid ambiguity Tinker) - 清晰相关信息示例避免歧义调整框架
- **RACE** (Role Action Context Example) - 角色行动上下文示例框架
- **RISE** (Role Input Steps Expectation) - 角色输入步骤期望框架
- **ROSES** (Role Objective Scenario Expected Solution Steps) - 角色目标场景预期解决方案步骤框架

### 任务执行框架
- **RTF** (Role Task Format) - 角色任务格式框架
- **SAGE** (Situation Action Goal Expectation) - 情况行动目标期望框架
- **SCOPE** (Scenario Complications Objective Plan Evaluation) - 场景复杂性目标计划评估框架
- **SPA** (Scenario Problem Action Result) - 场景问题行动结果框架
- **TAG** (Task Action Goal) - 任务行动目标框架

### 高级结构框架
- **TRACE** (Task Request Action Context Example) - 任务请求行动上下文示例框架
- **LangGPT** - LangGPT结构化提示词框架
- **Google** - Google提示词工程最佳实践框架
- **ICIO** (Instruction Context Input Data Output Indicator) - 指令上下文输入数据输出指标框架
- **Base** - 基础格式框架

## 🚀 使用方法

1. **选择框架**：根据你的思维习惯和项目需求选择合适的框架
2. **选择版本**：根据需要的详细程度选择完整版或轻量版

4. **添加需求**：将你的自动化测试需求文档附加到提示词后
5. **执行生成**：在AI助手中执行，获得定制化的自动化测试方案
