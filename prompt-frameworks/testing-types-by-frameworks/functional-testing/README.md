# 功能测试提示词框架

本目录包含20个不同框架的功能测试提示词，每个框架都有完整版(Full)和轻量版(Lite)两个版本。

## 🎯 功能测试核心内容

### 专业角色
- **资深功能测试专家**：10年以上功能测试经验
- **精通领域**：黑盒测试方法、功能分解测试、数据流测试

### 测试分类
- **核心业务功能测试**：关键业务流程验证
- **用户界面功能测试**：UI交互和显示验证
- **数据管理功能测试**：数据处理和存储验证
- **系统集成功能测试**：模块间集成验证

### 覆盖维度
- **功能覆盖**：正向流程、异常流程、边界条件
- **数据覆盖**：有效数据、无效数据、边界数据
- **用户角色覆盖**：不同权限用户的功能验证
- **环境覆盖**：不同环境下的功能表现

### 输出格式
功能测试方案包含：
- 测试概述
- 功能模块分析
- 测试策略设计
- 功能测试用例
- 集成功能测试
- 执行计划
- 验收标准

## 📁 文件命名规范

- **完整版**：`[框架名]-Full.md`
- **轻量版**：`[框架名]-Lite.md`

## 🚀 使用方法

1. 选择适合的框架（根据项目复杂度选择完整版或轻量版）
2. 复制对应框架文件中的提示词内容
3. 粘贴到AI助手中
4. 附加你的功能需求文档
5. 开始功能测试设计

## 💡 框架选择建议

### 完整版适用场景
- 复杂业务系统的功能测试
- 需要详细测试策略的项目
- 多模块集成的功能验证
- 高质量要求的功能测试

### 轻量版适用场景
- 简单功能的快速测试
- 敏捷开发中的功能验证
- 原型系统的功能测试
- 时间紧迫的测试任务
