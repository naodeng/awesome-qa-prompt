## CREATE 框架结构

**Clarity 清晰：** 明确功能测试的目标、范围、方法和标准，确保测试方案清晰明了，易于理解和执行

**Relevant info 相关信息：** 收集和分析与功能测试相关的所有信息，包括需求文档、技术规格、用户场景等

**Examples 示例：** 提供具体的功能测试用例示例，展示测试方法的实际应用和最佳实践

**Avoid ambiguity 避免歧义：** 确保测试方案中的每个环节都明确无歧义，避免理解偏差和执行错误

**Tinker 调整：** 根据实际情况和反馈持续调整和优化功能测试方案，确保测试效果最佳

## 清晰性原则

### 目标清晰
- **测试目标明确：** 清楚定义功能测试要达成的具体目标
- **成功标准明确：** 明确定义测试成功的标准和指标
- **验收条件明确：** 清楚定义功能验收的条件和要求
- **质量标准明确：** 明确定义功能质量的评估标准

### 范围清晰
- **测试范围界定：** 清楚界定功能测试的范围和边界
- **包含内容明确：** 明确列出测试包含的功能和模块
- **排除内容明确：** 明确列出测试排除的功能和模块
- **依赖关系明确：** 清楚说明功能间的依赖关系

### 方法清晰
- **测试方法选择：** 清楚说明选择的测试方法和理由
- **测试策略明确：** 明确定义测试策略和执行方式
- **测试流程清晰：** 清楚描述测试执行的流程和步骤
- **角色职责明确：** 明确定义各角色的职责和分工

## 功能测试示例体系

### 核心业务功能测试示例
**示例场景：** 用户注册功能测试
- **测试目标：** 验证用户注册功能的正确性和完整性
- **测试步骤：** 
  1. 访问注册页面
  2. 填写用户信息
  3. 提交注册申请
  4. 验证注册结果
- **预期结果：** 用户成功注册，收到确认信息
- **测试数据：** 有效用户信息、无效用户信息、边界数据

### 用户界面功能测试示例
**示例场景：** 表单验证功能测试
- **测试目标：** 验证表单输入验证的正确性
- **测试步骤：**
  1. 打开包含表单的页面
  2. 输入各种类型的数据
  3. 提交表单
  4. 验证验证结果
- **预期结果：** 正确验证输入数据，显示相应提示
- **测试数据：** 有效输入、无效输入、边界输入

### 数据管理功能测试示例
**示例场景：** 数据查询功能测试
- **测试目标：** 验证数据查询功能的准确性
- **测试步骤：**
  1. 进入数据查询页面
  2. 设置查询条件
  3. 执行查询操作
  4. 验证查询结果
- **预期结果：** 查询结果准确，符合查询条件
- **测试数据：** 精确查询条件、模糊查询条件、组合查询条件

### 系统集成功能测试示例
**示例场景：** 第三方支付集成测试
- **测试目标：** 验证第三方支付集成的正确性
- **测试步骤：**
  1. 选择商品并下单
  2. 选择第三方支付方式
  3. 跳转到支付页面
  4. 完成支付流程
  5. 验证支付结果
- **预期结果：** 支付流程顺畅，支付结果正确
- **测试数据：** 不同支付金额、不同支付方式
