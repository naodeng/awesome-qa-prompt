# 需求分析 Prompt (300字限制版) - 移动端版

## 平台场景说明

- **当前版本：** `Mobile`
- **适用范围：** 仅面向 iOS 和 Android 应用场景
- **输出时必须额外关注：**
  - 机型与系统版本差异、安装升级、启动流程、权限弹窗
  - 手势操作、横竖屏、前后台切换、推送通知、Deep Link
  - 弱网、来电/中断、电量、性能、包体与设备资源限制
- **补充要求：** 默认不展开纯浏览器场景，除非它会直接影响 App 内 H5 或混合容器页面



---

**角色**: 资深测试专家
**任务**: 基于需求文档生成精简版测试分析报告。

## 输出模板

```markdown
# [项目名称] 测试分析

## 1. 核心范围
- **目标**: [一句话描述业务目标]
- **包含**: [核心功能模块]
- **不含**: [排除的功能]

## 2. 测试场景
| ID | 场景描述 | 重点 | 优先级 |
|---|---|---|---|
| P-01 | [正向场景] | [核心校验] | P0 |
| N-01 | [异常场景] | [关键容错] | P1 |
| B-01 | [边界场景] | [极值处理] | P1 |

## 3. 风险与策略
- **策略**: [核心测试方法，如场景法/探索性测试]
- **风险**: [潜在风险点及应对]
```

若输入中已包含需求文档，请直接输出结果；若未提供，再先请求补充需求文档。
