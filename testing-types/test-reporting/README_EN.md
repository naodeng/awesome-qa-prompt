# Test Reporting

English | [简体中文](./README.md)

## Module Overview

The Test Reporting module provides comprehensive test reporting guidance, helping testing teams create clear, insightful, and actionable test reports to communicate test results, quality status, and risks to stakeholders.

## Core Features

### 📊 Complete Report Structure
- **Executive Summary:** High-level overview for stakeholders
- **Test Metrics:** Quantitative test execution data
- **Quality Assessment:** Overall quality evaluation
- **Risk Analysis:** Identified risks and mitigation strategies

### 🎯 Key Metrics
- **Test Coverage:** Requirements and code coverage
- **Test Execution:** Pass/fail rates, execution progress
- **Defect Metrics:** Defect density, severity distribution
- **Trend Analysis:** Quality trends over time

### 🔍 Visualization
- **Charts and Graphs:** Visual representation of metrics
- **Dashboards:** Real-time quality dashboards
- **Trend Lines:** Historical data visualization
- **Heat Maps:** Risk and coverage visualization

### 🌐 Report Types
- **Daily Reports:** Daily test execution status
- **Sprint Reports:** Sprint-level quality summary
- **Release Reports:** Release readiness assessment
- **Executive Reports:** High-level quality overview

## Version Selection Guide

- `Standard-version/`: the default recommendation for most real-world use cases
- `*_Lite.md`: use when the input is limited or you want a faster output
- `*_Lean.md`: use only when a module provides a Lean variant and you want a more restrained result
- `CRISPE/RISE/ICIO/ROSES/LangGPT`: use only when you explicitly want that prompting framework

## Usage Guide

### Quick Start

1. **Select Prompt File**
   - Full Version: Comprehensive test reporting with detailed analysis
   - Lite Version: Quick test status reporting with key metrics

2. **Prepare Input Materials**
   ```
   Test Results: [Test execution results and metrics]
   Defect Data: [Defect tracking data]
   Coverage Data: [Test coverage information]
   Timeline: [Project timeline and milestones]
   ```

3. **Get Professional Test Report**
   - Well-structured report format
   - Clear metrics and visualizations
   - Actionable insights and recommendations
   - Stakeholder-appropriate content
