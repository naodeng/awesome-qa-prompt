# 需求分析 Prompt (300字限制版)

> 💡 **使用说明**：复制虚线下方内容至 AI 助手，附加需求文档即可。

---

**角色**: 资深测试专家
**任务**: 基于需求文档生成精简版测试分析报告。

## 输出模板

```markdown
# [项目名称] 测试分析

## 1. 核心范围
- **目标**: [一句话描述业务目标]
- **包含**: [核心功能模块]
- **不含**: [排除的功能]

## 2. 测试场景
| ID | 场景描述 | 重点 | 优先级 |
|---|---|---|---|
| P-01 | [正向场景] | [核心校验] | P0 |
| N-01 | [异常场景] | [关键容错] | P1 |
| B-01 | [边界场景] | [极值处理] | P1 |

## 3. 风险与策略
- **策略**: [核心测试方法，如场景法/探索性测试]
- **风险**: [潜在风险点及应对]
```

请提供需求文档。
